# Projet : Bannière de Consentement aux Cookies 🍪

## 📝 Description
Ce projet est l'une de mes toutes premières réalisations en développement web (HTML, CSS et JavaScript). L'objectif était de créer une bannière de cookies interactive, fonctionnelle et esthétique, tout en apprenant les bases de la manipulation du DOM et du stockage local.

Ce projet a été réalisé en suivant les instructions de [Roadmap.sh - Cookie Consent](https://roadmap.sh/projects/cookie-consent).

## 🚀 Fonctionnalités
- **Interface fixe** : La bannière reste visible en bas de l'écran pendant le défilement.
- **Interactivité** : Boutons "Accepter" et "Refuser" fonctionnels.
- **Persistance des données** : Utilisation du `localStorage` pour mémoriser le choix de l'utilisateur. Si l'utilisateur a déjà fait son choix, la bannière ne s'affiche plus lors des visites suivantes.
- **Design Responsive** : Utilisation de Flexbox pour un alignement propre des éléments.

## 🛠️ Technologies utilisées
- **HTML5** : Structure de la page et de la bannière.
- **CSS3** : Mise en forme, gestion de la position `fixed` et design des boutons.
- **JavaScript (Vanilla)** : Logique de clic, gestion du `localStorage` et modification dynamique des styles CSS.

## 📂 Structure des fichiers
- `index.html` : La structure principale du site.
- `web.css` : Le design et la mise en page.
- `web.js` : La logique de fonctionnement.
- `cookie.avif` : L'illustration utilisée pour la bannière.

## 💡 Ce que j'ai appris
En réalisant ce projet, j'ai pu mettre en pratique :
1. Le positionnement des éléments avec CSS (notamment `position: fixed`).
2. L'écoute d'événements en JavaScript (`addEventListener`).
3. La différence entre les types de données (String vs Boolean) lors de l'utilisation du `localStorage`.
4. L'importance de la structure des fichiers pour un déploiement réussi sur GitHub Pages.
